## 2. 遇到了什么问题？

1. **真实数据待交接**（导师先前已确认**有**）：真实管网 `.inp` + 余氯监测数据可用，但我方尚未拿到文件；Week 2–3 先用 Net1 **合成观测**跑通 T3 pipeline，**论文主案例以真实数据为准**；需确认数据格式、监测节点/时段及交接时间。
2. **0.2 mg/L 依据需拍板**：**主来源 = F2（WHO）** — GDWQ 4th ed + 氯背景文件 [WHO/SDE/WSH/03.04/45](https://cdn.who.int/media/docs/default-source/wash-documents/wash-chemicals/chlorine.pdf) 推荐管网末端 **free chlorine ≥ 0.2 mg/L**（健康 GV 为 **5 mg/L**，与 operational 阈值不同）。**F1（UK SI 2016/614）** 仅规定法定消毒义务 + residual disinfectant 监测（Reg 26、Schedule 3），**Schedule 1 无数值限值** — 英国语境下作监管背景引用，不单独支撑 0.2 mg/L 数字。
3. **观测误差只能先验占位**：D2 支持 DPD **σ≈0.02 mg/L** + 方法间 10–47% 系统差；无项目仪器 QA 记录。
4. **Net1 规模偏小**：9 节点，适合 demo / pipeline 调试；**论文主案例 = 真实管网**（导师已确认）；Net3/BWSN 仅作方法对照或附录。
5. **校准可识别性**（文献已提示）：C2 指出正常工况下多组 $k_b$、$k_w$ 可拟合同观测 — 支持本课题做 **不确定性感知** 而非仅 deterministic RMSE。
6. **校准顺序与方法学困惑**：plan 分 T4（先拟合 bulk/wall）与 T5（后加 sensor uncertainty）；但若 sensor 本身有偏差或噪声，把观测当「真值」去校准 decay 参数，是否会**把 measurement error 吸收进 $k_b$、$k_w$**？需确认：T4 是否应从第一天就在似然里加入 $\varepsilon$（WLS，$\sigma$ 来自 D2/D5 文献先验），还是 T4 纯 RMSE 仅作 debug baseline、主结果从 T5 开始。
7. **仓库同步**：本地 2 commits + 笔记「可参考要点」修改尚未 push。
