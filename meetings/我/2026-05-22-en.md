## 2. Issues Encountered

1. **Real data pending handover** (supervisor previously confirmed **yes**): real network `.inp` + chlorine monitoring are available, but files not yet received; use Net1 **synthetic observations** in Week 2–3 to run the T3 pipeline; **main thesis case = real data**; need format, monitoring nodes/period, and handover timeline.
2. **0.2 mg/L basis needs sign-off:** **Primary source = F2 (WHO)** — GDWQ 4th ed + chlorine background doc [WHO/SDE/WSH/03.04/45](https://cdn.who.int/media/docs/default-source/wash-documents/wash-chemicals/chlorine.pdf) recommends **free chlorine ≥ 0.2 mg/L** at the network extremity (health GV **5 mg/L** is separate from this operational threshold). **F1 (UK SI 2016/614)** only sets legal disinfection duty + residual-disinfectant monitoring (Reg 26, Schedule 3); **Schedule 1 has no numeric chlorine limit** — cite F1 for UK regulatory context, not for the 0.2 mg/L figure alone.
3. **Observation error can only be prior placeholders for now:** D2 supports DPD **σ≈0.02 mg/L** plus 10–47% systematic bias across methods; no project-specific instrument QA records yet.
4. **Net1 is small:** 9 nodes — fine for demo / pipeline debug; **main thesis case = real network** (supervisor confirmed); Net3/BWSN only if needed for method comparison.
5. **Calibration identifiability** (flagged in literature): C2 notes multiple ($k_b$, $k_w$) pairs can fit the same observations under normal operation — supports **uncertainty-aware** work rather than deterministic RMSE alone.
6. **Calibration order / methodology confusion:** The plan splits T4 (fit bulk/wall first) and T5 (add sensor uncertainty later). If sensors are biased or noisy, treating observations as “truth” when calibrating decay parameters may **absorb measurement error into $k_b$ and $k_w$**. Need sign-off: should T4 include $\varepsilon$ in the likelihood from day one (WLS, $\sigma$ from D2/D5 literature priors), or is T4 pure RMSE only a debug baseline with main results starting at T5?
7. **Repo sync:** 2 local commits + “project takeaways” note edits not yet pushed.
