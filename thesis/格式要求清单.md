## 4. 字数与论文结构

### 4.1 字数（项目硬约束）

`[P]` 论文正文上限 **12,000 words**（README §1）
`[T]` 计数口径未确认：是否含 abstract / table of contents / references / appendix。

**应对**：先按「正文 ≤ 12,000 words，不计 abstract / refs / appendix」做规划；2.1 Handbook 拿到后立即对齐。

### 4.2 推荐章节结构（与 README §7 一致）

| 章节 | 建议字数占比 | 关键问题 |
| --- | --- | --- |
| Title page | — | 含模块代码、题目、姓名、学号、导师、提交日期 |
| Abstract | ≤ 300 词 (`[R]`) | 动机 / 方法 / 关键结果 / 结论一段话讲清 |
| 1. Introduction | ~10–15% | 为什么余氯不确定性建模重要？研究问题与贡献 |
| 2. Background / Literature Review | ~15–20% | chlorine decay 机理 + EPANET/WNTR + 校准 + 不确定性方法 |
| 3. Methodology | ~20–25% | 网络模型 + 数据 + 参数 + 误差模型 + MC / Bayesian 流程 |
| 4. Results | ~20–25% | 仿真 + 校准对比 + 不确定性输出 |
| 5. Discussion | ~10–15% | 工程意义、不确定性影响、局限 |
| 6. Conclusion | ~5% | 回答每个研究问题 + 未来工作 |
| References | 不计入（`[T]` 待确认） | Vancouver 或 Harvard，全文统一 |
| Appendix | 不计入（`[T]` 待确认） | 代码片段 / 附加图表 / 推导细节 |

`[R]` Abstract、Introduction、Conclusion 必须能让**本领域以外的合格专业人士**读懂（Imperial PGT 通用写作建议明文要求）。

## 7. Generative AI 使用披露

`[O]` Imperial 官方政策（[Generative AI guidance](https://www.imperial.ac.uk/admin-services/library/learning-support/generative-ai-guidance/)）：

> Unless explicitly authorized by your department, using AI to create assessed work may be treated as academic misconduct.

具体要求：

- 必须包含一份**声明**，列出：
  - AI 工具**名称 + 版本**（例如 `Claude Sonnet 4.5`、`ChatGPT-5`）
  - **出版者**（例如 `Anthropic`、`OpenAI`）
  - **URL**
  - **用途简述**（"用于检查代码语法 / 润色英文 / 整理引用格式"）
  - **声明"工作仍是我自己的"**
- 先看**院系层面规则**，CEE 可能在 Handbook 里有进一步限制。
- 不能把 AI 生成段落直接交。

`[P]` 项目层面（README §12）已经写过：AI 不能替代文献阅读、模型判断、结果解释；论文里必须按学院要求披露。

`[T]` CEE 是否有自己的 AI 披露模板 / 附录格式？→ 问导师。

## 9. 提交前最后核对清单（按顺序逐条勾）

提交日期前 **48 小时** 开始执行下面这一组检查：

### 9.1 文件本身
- [ ] 文件已导出为 **PDF/A-2b 或 -3b**
- [ ] 用 [veraPDF](https://verapdf.org/home/) 或 Adobe Preflight 验证通过
- [ ] 所有字体在 Adobe Reader → File → Properties → Fonts 显示 `Embedded` 或 `Embedded Subset`
- [ ] 打开 PDF 看封面、目录、页码、所有图表都正常显示

### 9.2 内容
- [ ] 正文字数 ≤ 12,000（按 Handbook 口径计算）
- [ ] Title / Abstract / Introduction / Background / Methodology / Results / Discussion / Conclusion / References / Appendix 章节齐全
- [ ] Abstract ≤ 300 词，且独立可读
- [ ] 每张图有编号 + caption + 数据来源
- [ ] 每张表有编号 + caption
- [ ] 引用风格统一（Vancouver 或 Harvard，二选一确认后全文统一）
- [ ] 每条参考文献都在正文出现过
- [ ] 每个正文引用都能在 References 找到
- [ ] AI 使用声明已写入（Acknowledgement 或独立 Declaration）

### 9.3 学术诚信
- [ ] Turnitin 已提交并查看过报告
- [ ] 自查相似度报告无未引用的高相似段落
- [ ] 所有非自己作品的图 / 表 / 数据都已标注来源
- [ ] Acknowledgement / Declaration 中列出导师、合作者、资助方

### 9.4 备份与提交
- [ ] 主文件 + 备份各存一份在两个不同位置（GitHub + 本地 + 云盘任选两个）
- [ ] 通过课程 / 学院规定的渠道提交（Blackboard / Turnitin / 邮箱，以 Handbook 为准）
- [ ] 收到提交确认邮件或截图
